No-Store, Cache-Busted, Short-Lived Assets

Every visit is a cold start. no-store on HTML, cache-busting query strings on JS/CSS (so browsers never reuse them), and max-age=60 on images mean every user — including returning ones — downloads everything from scratch.

❌ What's wrong

  • no-store on HTML — browser deletes the response immediately after rendering. Every visit downloads the full HTML document again.
  • JS/CSS loaded with ?v=timestamp query — the filename changes every deploy, so CDN can't cache them permanently. Also served with no-store.
  • Images cache for only 60 seconds — a user who reads an article for 2 minutes will re-download all images on the same page.
  • No service worker — offline visits or flaky networks get nothing.

✅ How to fix → good.html

  • HTML: no-cache (not no-store) + ETag — browser revalidates but gets 304 if unchanged (zero bytes).
  • JS/CSS: content-hash filenames (app.a3f9c2.js) + max-age=31536000, immutable — cached forever, invalidated by filename change.
  • Images: max-age=86400, stale-while-revalidate=604800 — cached 1 day, stale for 7 days while refreshing in background.
  • Service worker: cache-first for static assets, stale-while-revalidate for API responses.
