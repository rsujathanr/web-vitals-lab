❌ What's wrong

  • Bootstrap (194KB) + Font Awesome (92KB) both render-blocking — browser cannot paint until both download and parse completely.
  • Ball uses left CSS animation — not GPU-composited. Forces layout recalc + paint every frame at 60fps = 60 forced layouts per second.
  • transition: all on cards animates every property including layout-triggering ones (width, height, padding).
  • filter: drop-shadow prevents GPU compositing — the entire layer must be repainted on hover.
  • 20+ unused rules loaded and parsed on every visit (real codebases have thousands).

✅ How to fix → good.html

  • Inline critical CSS — zero blocking requests, FCP is immediate.
  • Load non-critical CSS via rel="preload" onload pattern — deferred without blocking.
  • Animate with transform: translateX() + will-change: transform — runs entirely on GPU compositor thread.
  • Replace transition: all with explicit properties: transition: border-color .2s, box-shadow .2s.
  • Use content-visibility: auto on off-screen sections — skip paint entirely until needed.
