Render-Blocking Scripts & Long Tasks

Two synchronous 800ms + 400ms busy-loops in <head> block HTML parsing entirely. A 2000ms synchronous task on button click destroys INP. Every keypress fires a 300ms blocking task.

❌ What's wrong

  • Two inline <script> in <head> with 800ms + 400ms busy-loops — HTML parser is completely blocked for 1.2s before a single pixel paints.
  • Button click runs a 2000ms synchronous while loop — the browser cannot process any input until it finishes (INP > 2000ms).
  • Search input fires a 300ms blocking task on every single keypress — typing feels like molasses.
  • No async or defer on any script — every script is a render blocker by default.

✅ How to fix → good.html

  • Remove all inline scripts from <head> — move work to defered external scripts.
  • On click: update UI immediately (instant visual feedback), then use scheduler.yield() before the heavy work.
  • Break large work into 50ms chunks separated by setTimeout(0) or scheduler.yield().
  • Debounce search input to 250ms — only one task fires after typing stops.
✕ The Bad Code
<!-- ✕ Render-blocking scripts in <head> --> <script> var s = Date.now(); while (Date.now() - s < 800) {} // blocks parsing! </script> <script> var s = Date.now(); while (Date.now() - s < 400) {} // still blocking </script> <!-- ✕ 2000ms long task on click → INP > 2000ms --> btn.addEventListener('click', function () { var s = Date.now(); while (Date.now() - s < 2000) {} // freezes browser updateUI(); }); <!-- ✕ 300ms block per keypress --> input.addEventListener('input', function () { var s = Date.now(); while (Date.now() - s < 300) {} // every. single. key. });
