FOIT, Unpreconnected Fonts, All 9 Weights

Google Fonts loads all 9 weights of Inter plus 6 weights of Playfair Display with display=block. No preconnect means extra DNS + TLS RTTs. Text is invisible until every font file downloads — that's FOIT.

Playfair Display — The quick brown fox

This paragraph uses Playfair Display with no fallback tuning. When the web font loaded, the text reflowed because Playfair's metrics differ from the generic serif fallback. That reflow is CLS.

❌ What's wrong

  • display=block: browser shows blank text (FOIT) until all font files download. On slow connections this can be 3–5 seconds of invisible content.
  • No preconnect to fonts.googleapis.com or fonts.gstatic.com — the browser discovers them late and pays DNS + TLS round-trips before fetching.
  • 9 weights of Inter = up to 9 separate woff2 network requests. Most pages use 2–3 weights. The others are pure waste.
  • No size-adjust or metric overrides on the fallback font — when Playfair swaps in, its different x-height causes line heights to change and text to reflow (CLS).

✅ How to fix → good.html

  • Use display=swap — text is always visible using the system font; web font swaps in when ready (FOUT, not FOIT).
  • Add preconnect to both Google Fonts servers to eliminate one RTT.
  • Load only 2 weights (400, 700) — covers most typographic needs with 7× fewer font files.
  • Add @font-face with size-adjust + ascent-override on the fallback so metrics match — CLS from the swap approaches zero.
