display=swap, Preconnect & Metric-Adjusted Fallbacks

Two weights only (400 + 700). display=swap keeps text visible at all times. Preconnect to both Google Font servers. Custom @font-face fallbacks with size-adjust and metric overrides to minimize CLS when the web font swaps in.

✅ What's fixed

  • display=swap — text is readable immediately using the fallback font; the web font swaps in silently when ready (FOUT not FOIT).
  • preconnect to fonts.googleapis.com and fonts.gstatic.com (with crossorigin) — DNS + TLS done before CSS is parsed.
  • Only 2 weights loaded — 7× fewer font file requests, much lower TTFB pressure.
  • @font-face with size-adjust: 107% + metric overrides on the Arial fallback for Inter — text barely moves when the font swaps.

📐 Before → After

  • 9 weights → 2 weights
  • display=block (FOIT) → display=swap (FOUT)
  • No preconnect → preconnect to both GFonts servers
  • No fallback tuning → size-adjust + metric overrides
  • Significant CLS on font swap → near-zero CLS
