❌ What's wrong

  • Hero is a CSS background-image — the HTML preload scanner cannot discover it, so it starts downloading only after CSS is parsed.
  • No width/height on product images — browser allocates 0px height, then reflowsafter each image loads (CLS spike).
  • No loading="lazy" — all 6 images download eagerly, competing with the LCP candidate for bandwidth.
  • No srcset — a 400px image on a 200px grid slot wastes 2× bandwidth.
  • No fetchpriority — the browser can't tell which image is the LCP candidate.

✅ How to fix → good.html

  • Replace CSS background with <img> + rel="preload" + fetchpriority="high".
  • Add explicit width and height attributes and aspect-ratio CSS on every image wrapper.
  • Add loading="lazy" + decoding="async" on below-fold images.
  • Add srcset for responsive sizes — serve 200px image in a 200px slot.
  • Add preconnect to picsum.photos to save one RTT.
