✅ What's fixed

  • <link rel="preload" as="image"> + fetchpriority="high" on the hero — browser fetches it at top priority before parsing the body.
  • Hero is an <img> — the LCP algorithm can measure it; a CSS background can't be an LCP candidate.
  • aspect-ratio: 4/3 wrapper on each card — exact space reserved before bytes arrive, CLS = 0.
  • loading="lazy" + decoding="async" on below-fold images — first two load eagerly for LCP, rest load on scroll.
  • preconnect to picsum.photos — DNS + TLS done before any image request fires.

📐 Before → After

  • CSS background-image hero → <img fetchpriority="high">
  • No dimensions → width/height attrs + aspect-ratio wrapper
  • All eager → first: eager, rest: loading="lazy"
  • No preload → <link rel="preload" as="image">
  • No preconnect → <link rel="preconnect"> in <head>
