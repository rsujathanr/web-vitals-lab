No preconnect, no dns-prefetch, no prefetch, no preload. Every cross-origin request pays the full DNS + TCP + TLS cost before a single byte of content arrives.

❌ What's wrong

  • No preconnect — browser discovers cross-origin servers only when it parses an HTML tag that needs them, paying DNS+TLS from zero each time.
  • No preload on the LCP image — browser discovers it only after parsing the CSS that references it.
  • No fetchpriority — browser allocates bandwidth equally across all requests; the LCP image competes with thumbnails.
  • No prefetch for likely next page — when the user navigates, they pay full load cost again.

✅ How to fix → good.html

  • preconnect to cross-origin servers in <head> — DNS+TLS completes before any tag references the origin.
  • preload as="image" on LCP candidate + fetchpriority="high".
  • dns-prefetch on analytics origins (lower priority than preconnect).
  • prefetch on the next likely navigation — loads it in idle time.
