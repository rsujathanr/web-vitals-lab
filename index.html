Web Vitals Lab

Eight interactive performance scenarios. Each shows a bad pattern and a good pattern side-by-side, with live Core Web Vitals measured in the browser using the web-vitals library.

≤200ms >600ms

Scenarios

🖼
Images
CSS background-image hero, images missing dimensions, no lazy loading, no srcset — the easiest way to tank LCP and cause layout shift.
LCP CLS TBT
✕ Bad ✓ Good
⚙️
JavaScript
Render-blocking scripts, synchronous busy-loops on startup and input events — the full recipe for poor FCP, TBT, and catastrophic INP.
FCP TBT INP
✕ Bad ✓ Good
🎨
CSS
Multiple render-blocking stylesheets, layout-thrashing animations, and hundreds of unused rules delay paint and cause jank every frame.
FCP LCP TBT
✕ Bad ✓ Good
✏️
Fonts
Loading all nine weights of a typeface with display=block makes text invisible for seconds. No preconnect means extra RTTs before any font byte arrives.
CLS FCP LCP
✕ Bad ✓ Good
📐
Layout Stability
Images with no dimensions, an ad slot that pops open 1.5 s after load, and a cookie banner that shoves the page up all conspire to spike CLS.
CLS
✕ Bad ✓ Good
🌐
Network
No preconnect, no resource hints, no priority signals — every origin pays the full DNS + TCP + TLS toll before a single byte arrives.
TTFB LCP FCP
✕ Bad ✓ Good
💾
Caching
Cache-busted filenames, no-store headers on static assets, and max-age=60 on images mean every visit is a cold start — returning users suffer the same as new ones.
TTFB LCP
✕ Bad ✓ Good
🔌
Third-Party
Sync analytics scripts in <head>, an eagerly-loaded YouTube iframe, and an immediately-injected chat widget block the main thread before the user sees anything.
TBT FCP LCP INP
✕ Bad ✓ Good

How to Measure

🔦
Lighthouse
Run a simulated load audit from DevTools → Lighthouse panel. Scores LCP, TBT (proxy for FID/INP), CLS, FCP, and Speed Index under throttled 4G.
Learn more ↗
📊
DevTools Performance
Record a trace to see the exact millisecond each frame paints, long tasks on the main thread, layout shifts, and the LCP candidate.
Learn more ↗
📄
PageSpeed Insights
Combines real-user data from the Chrome User Experience Report (CrUX) with a lab Lighthouse run — see your field vs lab scores side by side.
Open PSI ↗
📦
web-vitals.js
The official library used in this lab. Call onLCP(), onINP(), onCLS() etc. to collect real-user metrics and send them to your analytics endpoint.
